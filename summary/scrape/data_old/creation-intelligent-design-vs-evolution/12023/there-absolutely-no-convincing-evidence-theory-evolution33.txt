The computer loaded each set into the array, fed in the two tones, looked at the outputs and tried to find some feature ... to begin with, that feature was anything that didn't look totally random. The 'fittest' individual in the first generation produced a steady five-volt output no matter what tone it heard. The least fit instructions were then deleted, the fit ones were copied and recombined and the process was repeated.
            <br/>
            <br/>
            What's interesting about the experiment is ... how the system homed in on a solution, and the remarkable nature of that solution. By the 220th generation, the fittest circuit produced outputs that were pretty much the same as the inputs, two waveforms of different frequencies. The same effect could have been obtained with no circuit at all, just a bare wire!
            <br/>
            <br/>
            By the 650th generation, the output for the low tone was steady, but the high tone still produced a variable output signal. It took until generation 2800 for the circuit to give approximately steady, and different signals for the two tones; only by generation 4100 did the odd glitch get ironed out, after which little further evolution occurred. [NB: the final solution was: high tone = 0V output, low tone = steady 5V output]
            <br/>
            <br/>
            The strangest thing about the eventual solution was it's structure. No human engineer would have invented it. Indeed, no human engineer would have been able to find a solution with a mere 100 logic cells. The human engineer's solution, though, would have been comprehensible - we would be able to tell a convincing 'story' about why it worked. For example, it would include a 'clock' - a circuit that ticks at a constant rate. That would give you a baseline to compare the other frequencies against. But you can't make a clock with 100 logic cells. The evolutionary system didn't
           </i>
           bother
           <i>
            with a clock. Instead, it routed the input signal through a series of loops. These presumably generated time-delayed and otherwise processed versions of the signals, which eventually were combined to produce the steady outputs. Presumably. Thompson described how it functioned like this: 'Really, I haven't the faintest idea how it works'.
            <br/>
            <br/>
            Amazingly, further study showed that only 32 of it's 100 logic cells were actually needed - the rest could be removed from the circuit without affecting it's behaviour. At first it looked as if five other logic cells could be removed - they were not connected electronically to the rest, nor to the input or output. However, if these were removed, the circuit ceased to work. Presumably these cells reacted to physical properties of the rest of the circuit other than electrical current - magnetic fields, say. Whatever the reason, Thompson's hunch that a real silicon circuit would have more tricks up it's sleeve than a computer simulation turned out to be absolutely right.
            <br/>
            <br/>
            The message for basic evolutionary theory is ... important. In effect, it tells us that ... an evolved system may 'work' without it being clear how it does whatever it does. It may not follow any 'design principle'. ... Instead, it can follow ... emergent logic.
           </i>
           "
           <br/>
           <br/>
           That would be a situation starting with random base code and with a remarkably unpredictable outcome.

For example, it would include a 'clock' - a circuit that ticks at a constant rate. That would give you a baseline to compare the other frequencies against. But you can't make a clock with 100 logic cells. The evolutionary system didn't [/i]bother
               <i>
                with a clock. Instead, it routed the input signal through a series of loops. These presumably generated time-delayed and otherwise processed versions of the signals, which eventually were combined to produce the steady outputs. Presumably. Thompson described how it functioned like this: 'Really, I haven't the faintest idea how it works'.
                <br/>
                <br/>
                Amazingly, further study showed that only 32 of it's 100 logic cells were actually needed - the rest could be removed from the circuit without affecting it's behaviour. At first it looked as if five other logic cells could be removed - they were not connected electronically to the rest, nor to the input or output. However, if these were removed, the circuit ceased to work. Presumably these cells reacted to physical properties of the rest of the circuit other than electrical current - magnetic fields, say. Whatever the reason, Thompson's hunch that a real silicon circuit would have more tricks up it's sleeve than a computer simulation turned out to be absolutely right.
               </i>
              </div>
             </div>
            </div>
           </div>
           <i>
            <br/>
            This is an analog to the clock code, not a test of evolution but of selection.
            <br/>
            <br/>
            The key to a correct simulation is setting up selection parameters which are analogous to those in reality. A useless protein is not selected for, a useful enzyme is.
            <br/>
            <br/>
            An accurate simulation would not start out with functionality nor would the selective system look for seeds of it, it would only care for functionality itself no matter how inefficient.
